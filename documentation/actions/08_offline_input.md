# Оффлайн-ввод и последующая отправка (прораб, стройконтроль, инспектор)
*Цель*: пользователь может работать без сети, а система корректно примет данные позже

1. Пользователь создает записи (работы/материалы/замечания/нарушения) без связи
2. Приложение сохраняет их локально с уникальным `event_id`
3. При появлении сети отправляет пачку событий
4. Сервер:
	- принимает события *идемпотентно* (повтор не создает дубль)
	- применяет их в правильном порядке
	- возвращает результат (что принято, что отклонено)

**Результат**: оффлайн-отправка происходит успешно, сервер принимает событие один раз (повторная отправка не создает дубль); возвращает ответ: что принято, а что нужно переписать
