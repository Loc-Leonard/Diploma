# Замечания
*Зачем*: стройконтроль вносит замечания по качеству работ со сроком устранения, с фото/доками/комментариями. **Важно**: можно вносить только при посещении, и фиксируются время+местоположение

*Что хранить*:
- `id`, `object_id`
- Автор (стройконтроль)
- Текст/описание
- Срок устранения `due_date`
- Статус:
	- `OPEN`
	- `FIX_SUBMITTED` (прораб отметил устранение)
	- `ACCEPTED` / `REJECTED`
- Файлы/фото/доки
- Гео + время создания (обязательная фиксация)

*Правила*:
- Создать замечание можно только "при посещении"
- Прораб загружает подтверждение устранения
- Стройконтроль принимает/отклоняет результат устранения
