# Материал (поставка + входной контроль)
*Зачем*: ключевая задача - оцифровать поставки, входной контроль и автоматизировать извлечение данных из ТТН (CV), с возможностью ручного ввода и прикреплением документов

Два уровня:
1. **MaterialDelivery** - факт поставки (событие)
2. **MaterialDoc** - документы к поставке (ТТН, паспорт качества, фото)

*MaterialDelivery - что хранить*:
- `id`, `object_id`
- Привязка к `WorkItem` (к какой строке спецификации/работ относится материал)
- Дата/время поставки
- Гео
- Кто внес
- Распознанные/введенные поля:
	- Наименование материала
	- Объем/количество
	- Единицы измерения
- Признак "введено вручную" / "получено CV"

*MaterialDoc - что хранить*:
- `id`, `delivery_id`
- Тип: `TTN`, `QUALITY_PASSPORT`, `PHOTO`, `OTHER`
- Ссылка на файл
- Метаданные файла

*Распознавание ТТН (CV)*:
- Статус `PENDING / DONE / FAILED`
- Извлеченные значения (json)

*Правила*:
- Прораб фотографирует ТТН и паспорт качества (если есть)
- Предлагается CV для распознавания ТТН и сохранения объема/наименования
- Обязательно оставить ручной ввод
