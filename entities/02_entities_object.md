# Объект (карточка объекта)
*Зачем*: это "контейнер" всего: полигон, состав работ, сетевой график, материалы, замечания, нарушения, посещения.

*Что хранить*:
- `id`
- Название/адрес/описание
- `polygon` (координаты области работ)
- Статус жизненного цикла
- Ответственные:
	- `customer_control_user_id`
	- `foreman_user_id`
	- `inspector_user_id`
- "инициализация":
	- чек-лист акта открытия (как данные)
	- акт открытия (файл PDF)

*Статусы*:
- `PLANNED` - есть в списке планируемых объектов
- `WAITING_INSPECTOR_CONFIRMATION` - стройконтроль инициировал/заполнил чек-лист/направил инспектору
- `ACTIVE` - инспектор подтвердил, объект "в работе"
- `FINISHED` - закрытие работы с объектом

*Правила*:
- Карточка объекта показывает: полигон + состав работ + сетевой график
- Активация делается когда наступает плановая дата начала работ
- Доступ прораба к объекту появляется после подтверждения активации инспектором и назначения прораба
